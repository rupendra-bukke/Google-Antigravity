# Releases Folder Guide

This folder stores one markdown file per production release.

Naming:
- `vYYYY.MM.DD-NN.md`
- Example: `v2026.03.07-01.md`

Minimum required fields in each release note:
1. Version
2. Date
3. Branches/commit references
4. Summary of changes
5. Validation checklist
6. Rollback reference

Use template:
- `RELEASE_NOTE_TEMPLATE.md`

Latest release file:
- `v2026.03.12-01.md`
