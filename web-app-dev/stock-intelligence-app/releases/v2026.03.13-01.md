- A permanent free-tier guardrail was added to core workflow documents.

## Changes

### Added
- New backend endpoint: `GET /api/v1/expiry-calendar`.
- Mandatory free-tier guardrail in:
  - `FLOW_QUICK_REF.md`
  - `BRANCH_DEPLOY_FLOW.md`
  - `RELEASE_RUNBOOK.md`

### Changed
- `ExpiryBanner` now consumes live backend expiry calendar instead of local weekday-only assumptions.
- `ExpiryZeroHeroPanel` now consumes the same calendar source.
- Fallback behavior updated:
  - NIFTY fallback: weekly Tuesday
  - BANKNIFTY fallback: last Tuesday of month
  - FINNIFTY fallback: last Tuesday of month
  - SENSEX fallback: weekly Thursday
- Expiry API caching/polling tuned for free-tier safety:
  - backend cache: 6 hours
  - frontend refresh: hourly

### Fixed
- Incorrect next-expiry display in scenarios where live provider fetch was unavailable.
- Mismatch between expiry banner and zero-hero panel caused by separate local fallback logic.
