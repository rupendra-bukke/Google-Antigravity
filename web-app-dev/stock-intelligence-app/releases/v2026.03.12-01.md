## Summary

- This release focuses on operational stability on free-tier infrastructure.
- Main goal: reduce backend memory pressure and repeated request load while keeping the current trading UX intact.
- Added watchlist MVP navigation/page and moved watchlist data retrieval to a batched backend API.

## Changes

### Added
- Watchlist MVP page in frontend.
- Sidebar navigation support for Watchlist.
- Backend batched watchlist endpoint:
  - `GET /api/v1/watchlist-snapshot`

### Changed
- Analyze and watchlist backend flow optimized for lighter payloads.
- Multi-timeframe fetch now supports skipping `1m` data where not required.
- OHLC data handling optimized (numeric downcast + capped frame length).
- Gemini output token cap reduced to reduce heavy responses.
- Frontend polling intervals increased and hidden-tab polling guarded.

### Fixed
- Reduced probability of Render memory unit exceed events caused by refresh bursts.
- Lowered load from multi-call watchlist fetch pattern by replacing with single batched call.
